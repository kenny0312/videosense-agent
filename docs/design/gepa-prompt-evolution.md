# 设计:GEPA 式 prompt 自动进化(eval 反馈驱动,阶段②立项)

> 状态:Proposal(立项,未动工) · 前置:evals 地基(已建,143 题确定性判分 + 失败简报)· 关联:docs/eval-system-and-layer0-plan.md(四阶段路线的阶段②)、prompt-constitution-lessons.md(三层记忆)

## 1. 为什么(动机与证据)

手工往 prompt 加规则是**利息递增的债**:

- **IFScale**(arXiv 2507.11538):20 个前沿模型全部随指令密度上升而遵循率下降,且**先来的指令吃香、后加的被稀释**(primacy bias)—— 每加一条规则都在稀释已有规则;
- 我们实测:每轮固定注入 ≈ 4.2k tok,其中**工具声明(1.85k)已是最大块**,lessons 13/15 只剩 2 个空位;
- E-batch(2026-07-08)是**最后一轮手工修**:同类行为失败再现时,不再手加,改走本设计。

**GEPA**(arXiv 2507.19457,ICLR 2026 oral):让 LLM 读【执行轨迹 + 文本化失败反馈】→ 反思 → 进化 prompt 候选,比 GRPO 平均高 6%(样本效率 35×)、比 MIPROv2 高 10%+。它要的两样东西我们都有了:**可判分 eval(143 题确定性尺子)+ 文本失败反馈(评测简报,含金标依据/工具轨迹/栽在哪把尺子)**。

## 2. 目标 / 非目标

**目标**:一个半自动循环 —— eval 失败简报进,prompt 候选 diff 出,回归门守住,人只审 diff。
**非目标**:
- 不做全自动上线(候选 diff 必须人审 —— prompt 是系统性格,不全托管);
- **宪法不进搜索空间**(byte-stable 缓存前提 + 身份/安全条款不交给优化器);
- 不追论文的完整 Pareto 树(我们语料是十几段 prompt,不是千级模块)。

## 3. 搜索空间(优化什么)

| 可变 | 说明 |
|---|---|
| lessons 各条文本 | 改写/合并/退役(预算 ≤15 由 test 强制) |
| 工具声明 planner_desc | 各工具的用途/触发判据措辞 |
| ~~宪法~~ | **锁死**(见非目标) |

## 4. 循环形态(一轮 = 一个 PR)

```
① 跑全量 eval(live)→ 失败简报(现成:evals/briefing.py)
② 反思器(flash/pro 一次调用):读简报 + 当前 prompt 各段
   → 按失败聚类提出 K 个【候选修改】(每个 = 针对某段的 diff + 为什么 + 预计影响哪些题)
③ 对每个候选:应用 → 只重跑【受影响题 + 必过题】(省 token)→ 记分
④ 择优合成一个 prompt 变更集 → 全量 eval 回归门(通过率不得降、必过题全过)
⑤ 产出 PR(diff + 每处修改的动机引用简报条目)→ 人审合并
```

- ②③④ 脚本化(evals 包内新模块 `evolve.py`),⑤ 人工;
- 预算护栏:单轮 ≤N 次候选评估(env 可配,默认 4 候选 × 受影响题子集);
- 反思器输出结构化(候选=JSON diff),坏输出丢弃 fail-open。

## 5. 触发条件(什么时候跑)

- 全量 eval 通过率跌破上轮,或某方面 <85%,或必过题失守 —— 且失败属**行为类**(prompt 可治),非代码 bug/评测 bug;
- 手工修同类失败**第二次**出现 = 强触发(E-batch 已用掉"第一次"额度)。

## 6. 风险

- **过拟合 eval**:优化器只见 143 题 → 规则可能特化。缓解:回归门含全部方面分 + 简报的"评测自身问题"类失败不进反思器;题库持续扩(挖题脚本已有);
- **奖励欺骗**:反思器学会讨好尺子措辞。缓解:尺子是确定性程序(非 AI 判分),欺骗面小;人审 diff 是最后闸;
- **成本**:一轮 ≈ 1 次全量 + K 次子集 ≈ 2-3 次全量成本。触发式而非例行,可控。

## 7. 里程碑

- **G0**:`evals/evolve.py` 骨架 —— 简报→反思器→候选 JSON(不跑评估,人肉看候选质量)
- **G1**:候选评估 + 回归门 + PR 产出(端到端一轮,人审)
- **G2**:两轮真实使用后复盘 —— 值不值得保留/调形态(数据说话,同 SA-3 惯例)
