# v1.4 · 前端会话历史侧栏(ChatGPT 式)
日期:2026-06-25 ｜ 类型:feature

主题:给聊天前端加一个左侧会话栏 —— 历史对话列表、一键新对话、点开旧对话继续聊。纯前端(`web/index.html`),后端零改:会话靠已有的 `session_id` 续聊,后端记忆照旧落库。

## 更新内容

### 会话历史侧栏
- 左侧栏列出历史对话(标题取首句),"＋ 新对话"开新会话,点条目重开并还原整段富渲染历史(表格/图表/DAG/SQL/trace),悬停出现 `×` 删除。
- 对话存在浏览器 **localStorage**(`vs_chats_v1`):`[{id, title, updatedAt, messages:[user|assistant(完整响应)]}]`。`id` 即后端 `session_id` —— 重开旧对话再发,后端按该 id 续上下文(meta/followup 照常解析)。
- 布局:`.shell` = 侧栏 + 主区;聊天列居中(≤860px);≤720px 侧栏改为浮层 + 半透明遮罩,新对话/选中后自动收起。

### 健壮性(经多 agent 对抗审查后修)
- **in-flight 守卫**:同一时刻只允许一个发送在途 —— 狂按 Enter 不再并发建出重复会话。
- **快照路由**:每次发送在发起时快照归属会话;响应严格回填到那条会话,而非 await 后的当前会话 —— 中途切换/新建/删除都不会错配、不会复活已删会话、不会劫持新草稿。
- **URL 协议白名单**:回放历史里的 `plot_url` 只放行 http/https(挡掉被篡改的 `javascript:`/`data:`)。
- **localStorage 配额**:超限时先裁最旧的非当前会话、再裁当前会话最旧消息,绝不静默丢当前会话或抛错。
- 发送后焦点回到输入框;sync `def` 端点的线程池并发已在 API 层每会话锁(另见会话后端改动)处理。

## 影响 / 注意
- API 契约不变;后端零改(仅 `web/index.html`)。
- 历史存浏览器本地,**不跨设备/浏览器**;清缓存即清历史。后端会话记忆仍有 TTL(默认 24h),过期后旧对话可继续提新问题,但跨轮指代("plot those")可能失效。
- 富渲染历史按整条 API 响应入库,localStorage 约 5MB 上限由上面的配额裁剪兜底。
