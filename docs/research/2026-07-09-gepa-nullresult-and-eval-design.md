# GEPA 空结论归因 + 业界/学术界评测数据集设计调研

> 2026-07-09 · 五路多智能体调研(1 路本地取证 + 4 路文献,链接均实际打开核实)
> 触发问题:GEPA 第一轮为何没改进?是题少、同模型自改、还是别的?顺带:业界/学术界的评测方式与数据集怎么获取和设计?

## 第一部分:GEPA 空结论——用数据定罪(不是猜)

取证扒到了第一轮的完整运行产物(`evals/gepa/runs/g0709-122306/state.json`,234KB 分数矩阵,被 gitignore)。

**第一轮实况**:val 100 题、记分板 n=1、反思模型 gemini-2.5-pro(被优化的 agent 主脑是 gemini-3.5-flash,**主路径不是同模型自改**)、实跑 2 代 2 候选、花 $13.34。

**主因排序(证据支撑)**:
1. **天花板(绝对主因)**:gen0 基线在 val 100 题上首考 0.9082,但**全新 rollouts 重考 = 1.0000,100/100 满分**。9 道首考失分全是温度抖动,重考全恢复。真实水平就是满分——**优化器数学上没有任何东西可优化**(sign-test 只在父本<1 的题上才可能记"赢")。
2. **n=1 噪声(放大器)**:基线 n=1 让 0.9082 冒充真实水平,给两个候选白送 9 个"胜局"(且 9/9 与基线倒霉的 9 题完全重合——赢的全是基线的坏运气)。制造两次 p=0.011 假显著,烧 ~$4.3 重考费去证伪。
3. **对称重考救了场**:候选 c1/c2 首考"击败"基线(0.98 vs 0.91),但双方全新 rollouts 重考后 c1 0.9865 vs gen0 1.0 → 0 胜 3 负,c2 同理 → **两个都被判无效**。没有这道"赢家诅咒"闸,本轮会带着假 +7.7pp 发 PR。
4. **同模型自改 ≈ 不成立**:反思器是 2.5-pro、被测是 3.5-flash;候选提案内容合理、精确引用病历。反思质量不是瓶颈。
5. **反馈太薄 = 不成立**:病历含问/答/判分标准/金标依据/完整工具轨迹,信息很厚。

**一句话因果链**:val 对当前 prompt 无难度(真满分)→ 唯一"失败信号"全是 n=1 采样噪声 → 反思器被迫追噪声 → 候选靠基线霉运假显著 → 对称重考归零。**空结论是题库和采样数的问题,不是引擎或反思模型的问题**。已定前置"题库上难度 + 基线 n≥2"完全对症。

## 第二部分:同模型自改进行不行(文献)

**答案分两半**:无外部反馈的纯自反思基本不行(Huang et al. ICLR 2024 "LLMs Cannot Self-Correct Reasoning Yet",Self-Refine 在 GSM8K 几乎不涨);**但有可靠外部判分反馈时,同款模型反思有效**——Reflexion(HumanEval 80→91%)、GEPA 论文本身就是"同款模型(甚至 8B)+判分反馈"的设置拿到 +12~14%。

- "用更强模型当反思器"是 DSPy/GEPA 官方**实践建议**(推荐 gpt-5),但**论文没有强弱消融**,反例(8B 自反思也涨)就写在正文。
- 我们有确定性判分器,不在 Huang 的否定射程内。**"同款模型"不是主嫌**;但下一轮可顺手把反思器换 Gemini Pro(被测保持 flash),便宜地排除这个变量。
- 注:同模型当**裁判**才有自偏袒风险(Panickssery NeurIPS 2024),我们判分器是确定性的,这条通道本来就关着。

GEPA 已知失败模式(与我们印证):①只给分数不给失败原因(反思无从下手);②瓶颈是领域知识/模型能力而非指令(SuperCoder 汇编优化,gpt-5 反思也救不了;GEPA 自己在 AIME 输给 GRPO);③反思模型太弱(Decagon:GPT-4o-mini 反思=prompt 原地不动);④训练样本太多→prompt 膨胀过拟合(Decagon:50→500 反而更差,甜点 20–100);⑤贪心选种不用 Pareto 会卡死。

## 第三部分:学术界数据集怎么设计(8 个基准的构建方法)

| 路线 | 代表 | 成本 | 质量保障 | 规模 |
|---|---|---|---|---|
| 人工专家出题+验证 | GAIA、HLE、OSWorld | 极高(OSWorld ~5 人时/题;HLE $50 万奖金) | 独立复答一致 / 两轮专家评审 / 非出题人实测 | 几百~2500 |
| 真实数据挖掘改造 | SWE-bench、AgentBench | 低(全自动过滤) | 差——**68% 有毛病**,必须补人工审 | 千级 |
| LLM 生成+人工过滤 | BFCL V3 | 中 | 多轮过滤(11 轮)+人工金标+单元测试 | 千级 |
| **仿真世界+程序化验证** | **τ-bench、τ²-bench**、WebArena | 建世界贵,出题边际成本低 | 判分天生确定(状态比对);τ² 用可验证原子件拼题,正确性来自构造本身 | 可程序化扩 |

**关键教训**:
- **SWE-bench Verified**:93 名开发者人工审,发现 68.3% 的题有毛病(38% 描述不清、61% 测试过严),清洗后 GPT-4o 分数翻倍——**原基准里近一半"模型失败"其实是题的失败**。
- **GAIA**:"每题两名新标注员独立作答、三人一致才入库"是最便宜有效的无歧义性检验。
- **τ²-bench**:组合式任务生成器——把可验证原子件程序化拼成复合题,金标随拼装自动生成。这是我们挖题脚本的演化方向。
- **HLE**:收题闸门=候选题先难倒当前最强模型,答对的不要。正好接我们"题库上难度"。
- **BFCL**:一定要有"库里根本没有此物"的相关性检测题(考 agent 知不知道该拒);判分用状态比对+子集匹配允许多条正确路径,避免过严。
- **pass^k 出处 = τ-bench 论文**(Yao et al. 2024):同题独立跑 k 次全部成功的概率,衡量可靠性下限(镜像 Codex 的 pass@k)。GPT-4o 在 τ-retail 上 pass^1≈50%、pass^8<25%。

**防污染**:私有仿真世界天然抗污染(答案取决于私有库状态);可埋金丝雀 GUID(BIG-bench)、留 held-out 私有哨兵集(GAIA/HLE)、时间窗口滚动(LiveBench 每月换血)。
**防饱和**:Verified 子集清噪 / 难度分层(GAIA L1-L3 分层报分)/ 动态题库 / LLM 预筛难度门。

## 第四部分:工业界怎么建(Anthropic / OpenAI / 实践派)

**Anthropic《Demystifying evals for AI agents》**:①别造题,从"发版前手动检查的行为 + 用户真实 bug/工单"挖,**20-50 道真实失败题够起步**;②好题=两个专家独立判会得同一结论;③**别按固定步骤判,判结局状态**(agent 常走设计者没料到的合法路径);④rubric 分维度独立判、留 Unknown 出口;⑤**持续读 transcript**——不读就不知道判分器好不好,分数停滞时读它分辨是 agent 差还是评测坏了。

**OpenAI**:生产日志+专家手写题混合;rubric 给具体档位例子;judge 先用最强模型、和人工对齐后再降级;pairwise 比绝对分稳;控长度偏见。GDPval:自动判分器与人一致率 66%、而**人-人一致率也才 71%**——judge 逼近人-人一致即合格。HealthBench:医生逐题写 rubric(48562 条判据)+ meta-eval(评判分器本身,达到医生互评水平才用)。

**LLM-as-judge 规范**:MT-Bench(GPT-4 judge 与人一致 >80% ≈ 人-人水平,别追 100%;位置/长度/自偏好三大偏见,pairwise 换序跑两遍);judge 别用同家模型(Panickssery);对表度量用 Cohen's κ 或二元 precision/recall(Eugene Yan);判据会随读判例漂移(criteria drift,Shankar),rubric 必须迭代。

**实践派(Hamel/Eugene Yan)**:看数据→错误归类→每类错写一个测试;**能用代码断言就别用 judge**;critique shadowing——专家对 ~30 例做二元 pass/fail+批注,迭代 judge 到一致率 >90%,但数据不平衡时看 precision/recall 别看一致率;通用 eval(MMLU 类)和你的应用基本不相关,必须自建任务特定评测。

## 对我们这套评测的行动清单(按性价比)

1. **收题难度闸门(HLE)**:扩容波次先跑当前 agent 基线 n≥2,答对的候选题不收——这既是 GEPA 下一轮的前置(制造真难题),也是防饱和的机械化手段。**一举两得**。
2. **独立复核协议(GAIA)**:单人自建的盲区是"自己出的题自然觉得没歧义"。替代方案:隔几天回头重判自己的题,不一致的删;或用对过表的裁判当"第二双眼睛"预筛。
3. **判分器防过严审计(SWE-bench Verified)**:抽样看被判错的答案有多少其实是对的——我们的"冤案复核"已是微缩版,可定期重跑。
4. **难度分层+私有哨兵集**:给 177 题打 L1/L2/L3 标签、报分层通过率;留 10-20 道不入库题当污染哨兵。
5. **judge 对表看 precision/recall 不只看一致率**;题库开源前埋金丝雀 GUID。
6. **组合式生成器(τ²)**:把假世界的可验证原子事实做成积木,程序化拼复合题,是挖题脚本的长期演化方向。
